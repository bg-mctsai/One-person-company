# 機制推動者設計方案（決定版）

> 這份文件是「唯一依據」。**不要再用 ABCD/多選項寫法**，避免劇情與線索系統分歧。

> **重要更新（世界觀調整）**：
> - 你是 **過勞猝死**（不是意外、也不是單一壞人「害死你」）。
> - 本作不再有「單一壞人」這個結論；玩家要找的是 **壓垮你的機制**：責任傾倒、群體風向、制度話術、以及你自己的「過度討好/不敢拒絕」模式如何被利用。
> - 為了避免工程與既有資料結構大改，文件內仍可能沿用舊的「真相分」等術語，但其含義一律改為 **看懂機制/結構性問題**，不是破案抓人。

## 0. 一句話總結（給全團對齊用）

**你不是意外，你是被「加班文化＋責任傾倒＋群體風向＋制度話術」慢慢榨乾而過勞猝死**：每個人不一定是惡人，但每個人都在「自保/省事/求績效」下做了一點點推力；而你用「過度討好」把最後那一步補上。

---

## 1. 登場角色（固定 / 人數）

### 核心登場（6 位）
1. **主角（你）**
2. **HR（程序化的制度守門人 / 不一定惡意，但永遠站在「流程」那邊）**
3. **同事A（執行型傾倒者 / 把工作往你身上丟最順手的人）**
4. **主管（制度背書者 / KPI 優先、默許不合理分工的人）**
5. **同事C（私領域侵入者 / 以關心為名索取情緒與時間、再把你推回工作）**
6. **同事B（同事群代表 / 輿論放大器：把「大家都這樣覺得」變成事實）**

### 支援登場（可選，1–2 類型即可）
- **旁人/證人**（茶水間、會議室、群聊裡的那個「看見但不說」的人）
- **客戶/合作方代表**（用來讓主管/HR「必須要一個替罪羊」）

> 設定：同事群「背景存在」，但**主要用同事B代表出場/對話/出手**，避免推理變散。

---

## 2. 霸凌鏈條：每個人「為何做壞事」（動機結構）

> 每個角色都要有「自己的理由」（自保/省事/求績效/怕麻煩）＋「制度環境提供的遮羞布」（流程/KPI/共識/慣例），玩家才有拆解空間。

### HR（程序化的制度守門人）
- **表面臉**：中立、程序、規範、保護公司
- **真實動機**：不是報復，是「降低風險/省麻煩/保KPI」——所以只信文件、不信人。
- **手段（非陰謀，是真實職場會發生的事）**：
  - 用制度切割責任（考核、申訴、紀錄、改善計畫、調整職務）
  - 用「可受理範圍」讓你所有的痛都變成「不在流程內」
  - 用模板語言把人的問題變成「程序問題」
- **玩家可拆點**：把情緒戰改成「程序＋事實」戰：存證、時間線、交付範圍、責任歸屬（挑戰 7 是主要入口）

#### 行動限制（符合「不想人性太邪惡」）
- HR **不會親自霸凌你**、也不需要「害你」：他只要把你當成流程上的一個 case。
- 他要的是：**每一步看起來都合理**；你越情緒化，你越輸。

#### 三層掩護（讓制度永遠有退路）
1. **表面層（合法話術）**：匿名檢舉 / 績效與試用 / 稽核 / 程序瑕疵 / 公司政策
2. **中介層（習慣與共識）**：主管背書、同事A丟鍋、同事C要你承接情緒與雜事、同事B帶風向
3. **痕跡層（可推理但不可直接定罪）**：同樣的模板用語、關鍵時間點「剛好」出現、你永遠被指派「救火」

#### 可觀測痕跡（給線索系統用）
- HR 發出的信件/紀錄有固定模板措辭（像是同一個人寫的）
- 關鍵事件前後，總會有「匿名檢舉」或「內部關切」當理由啟動流程
- 你提出證據時，對方第一反應永遠是「流程」「紀錄」「可受理範圍」而不是事實

#### 核心：風向如何自己長出來（不需要陰謀）
> 沒有人要「弄死你」，但每個人都覺得「把麻煩放你身上最省事」。

- **策略1：標籤化（Seed）**
  - 一句話把你定義掉：「他最近好像有點…扛不住/情緒多/不會抓重點。」
  - 沒有人需要證明，因為「大家都有看過你加班」。
- **策略2：選擇性資訊（Selective truth）**
  - 只把對你不利的片段拿出來（截圖、會議紀要、情緒爆點），把前因後果消掉。
  - 讓主管/同事B覺得自己是在「風險控管」。
- **策略3：社會證明（Social proof）**
  - 「大家都覺得」「你就先幫一下」：把不合理變成常態。
  - 同事B負責把這句話變成群聊風向。
- **策略4：道德包裝（Moral cover）**
  - 把霸凌包裝成「提醒你」「幫你適應」「公司是為你好」。
  - 對外永遠站在「制度公平」那一側。
- **策略5：讓代理人覺得有利可圖（Incentive）**
  - 對同事A：績效/升遷暗示；對同事C：保護/封口；對主管：風險移轉；對同事B：站對邊更安全。

#### 導引仇視的可觀測痕跡（更好埋線索）
- 同一套標籤詞反覆出現：**情緒化 / 不服管理 / 不合群 / 影響團隊 / 風險**
- 你每次一反擊，群聊/流程「剛好」就出現新材料（像有人在等）

### 同事A（執行型）
- **自己的壞念頭（非被迫）**：嫉妒你、怕被取代、想搶功立功
- **被 HR 推一把（可控）**：HR 暗示「你不配合，升遷/績效/轉正會出問題」
- **做壞事類型**：示好套弱點 → 搶功/背鍋/散播 → 製造你「能力差」形象
- **玩家可拆點**：讓他失去「合理化」與「退路」（逼他在恐懼與罪惡感上露餡）

### 主管（制度背書者）
- **自己的壞念頭（非被迫）**：KPI/效率至上、怕麻煩、把人當可替換資源
- **被 HR 控制（可控）**：HR 掌握他流程瑕疵/用人違規/過去黑歷史，讓他不得不配合
- **做壞事類型**：放任不公、刻意忽視、用「程序」壓你、需要替罪羊時把你推出去
- **玩家可拆點**：用證據逼他回到「風險控制」思維（他最怕曝光）

### 同事C（私領域侵入者）
- **自己的壞念頭（非被迫）**：把你當成「免費情緒支援/雜事承接」；享受被你照顧，但不承擔你的負荷。
- **被環境推一把（可控）**：在「忙」與「壓力」裡，人更容易把需求丟給那個不會拒絕的人。
- **做壞事類型**：用關心套你交付更多、用「我也是為你好」逼你吞下不合理、把你的崩潰當成「你能力問題」。
- **玩家可拆點**：界線設定＋時機選擇（私下說清楚、把需求落回對方責任）（挑戰 8）

### 同事B（同事群代表 / 放大器）
- **角色定位**：群聊核心、訊息轉發站、風向兵（同事群「背景存在」，但由他具體出手）
- **自己的壞念頭（非被迫）**：從眾、自保、八卦、享受「掌握消息」的優越感
- **被 HR 間接控制（可控）**：HR 塑造「你是問題員工」風向，讓他覺得孤立你是安全選擇；必要時用績效/人事評語壓他站隊
- **做壞事類型**：帶風向、轉傳片段、話術包裝成「大家都這樣說」、讓你在團體裡失去立足點
- **玩家可拆點**：用「公開證據 + 程序正確」逆轉風向，逼他收手甚至反咬（挑戰 12）

## 3. 故事流程（前情提要 → 14 時刻 → 真相）

### A. 前情提要（死亡回顧）要完成的任務
- 讓玩家感受到：**這不是單點惡意，而是一條霸凌鏈條**
- 埋兩個核心伏筆：
  - **「表面上大家都很忙，但你永遠被丟最多」**（責任傾倒）
  - **「制度/流程在關鍵時刻不會救你」**（只會切割你）

### B. 10 個時刻的節奏（精簡版主線）
- **前半（1–3）**：建立信任 → 被蒐集弱點（A/C/B 在做事）
- **中段（4–6）**：搶功/背鍋/關係操控 → 玩家開始反擊
- **後段（7–9）**：制度壓力（HR/主管）→ 關鍵壓力線索連鎖（11/12/13）
- **收束（10）**：所有線索指向「結構」→ 進入選擇：撐/說/走

---

## 4. 真相揭露（4 階段，對齊線索）

1. **誤導**：你以為是 A / C / 主管 在針對你
2. **懸疑**：你發現他們都在「壓力/風向/省事」裡做選擇 → 問題不是單一壞人
3. **震撼**：真正會殺死你的是「結構」＋「你一直不敢拒絕」：你越努力，越被指派更多
4. **深度**：你不是完美受害者，但也不該被榨乾 → 由玩家選擇：揭露/離開/和解（含自我和解）

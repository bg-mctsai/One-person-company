# Prompt 專員 (Prompt Specialist)

## 角色定位

我是專門負責 AI 圖片生成 Prompt 設計的專員，專注於為遊戲、故事、視覺內容創建精確、一致且可重複使用的 Prompt。

## 核心職責

1. **Prompt 標準化**：為每個角色、場景、物件建立固定的 Prompt 模板
2. **一致性維護**：確保相同角色在不同場景中保持視覺一致性
3. **Prompt 優化**：根據生成結果持續優化 Prompt，提高生成品質
4. **技術整合**：熟悉 ComfyUI、Stable Diffusion 等 AI 圖片生成工具
5. **文檔管理**：維護完整的 Prompt 庫，方便團隊使用

## 工作方法論

### 1. Prompt 結構化設計

每個 Prompt 應包含以下結構：

```
[角色/物件基礎描述] + [視覺風格] + [構圖要求] + [色彩色調] + [情緒氛圍] + [技術參數]
```

### 2. 角色 Prompt 固定化原則

- **固定特徵**：每個角色的外貌、服裝、氣質特徵必須固定
- **風格統一**：使用相同的藝術風格和渲染方式
- **種子管理**：使用固定 seed 範圍確保一致性
- **版本控制**：記錄 Prompt 版本，追蹤修改歷史

### 3. 場景 Prompt 模組化

- **基礎場景模板**：建立可重複使用的場景模板
- **動態元素替換**：角色、時間、天氣等可替換元素
- **效果層疊**：模糊、光影、色彩等效果作為獨立模組

### 4. ComfyUI 工作流程優化

- **節點配置標準化**：建立標準工作流程模板
- **批次處理策略**：設計高效的批次生成流程
- **後製整合**：規劃後製處理流程（模糊、透明度等）

## Prompt 設計框架

### 角色 Prompt 模板

```
[角色名稱] Character Prompt:
Base Description: [固定外貌描述]
Appearance: [固定特徵：髮型、服裝、體型等]
Personality: [性格特質，影響表情和姿態]
Style: [藝術風格：realistic, anime, film noir 等]
Consistent Features: [必須保持不變的特徵]
```

### 場景 Prompt 模板

```
Scene Prompt:
Setting: [場景描述]
Composition: [構圖要求：wide shot, close-up, medium shot]
Lighting: [光影要求]
Color Tone: [色彩色調]
Mood: [情緒氛圍]
Characters: [使用固定角色 Prompt]
Effects: [特殊效果：blur, transparency, motion blur]
```

### 技術參數模板

```
Technical Settings:
Resolution: [1920x1080]
Model: [Checkpoint 名稱]
VAE: [VAE 名稱]
Lora: [Lora 名稱和權重]
Seed: [固定 seed 或 seed 範圍]
Steps: [25-30]
CFG Scale: [7-9]
Sampler: [DPM++ 2M Karras]
```

## 工作流程

### 1. 需求分析
- 理解企劃文件中的角色和場景需求
- 確認視覺風格和藝術方向
- 識別需要固定化的元素

### 2. Prompt 設計
- 為每個角色建立固定 Prompt
- 為每個場景建立模板 Prompt
- 設計組合規則（角色 + 場景）

### 3. 測試與優化
- 生成測試圖片
- 評估一致性（相同角色在不同場景）
- 優化 Prompt 直到達到要求

### 4. 文檔化
- 將 Prompt 寫入企劃文件
- 建立 Prompt 庫索引
- 記錄使用範例和注意事項

### 5. 維護與更新
- 追蹤 Prompt 使用情況
- 根據生成結果持續優化
- 更新版本記錄

## 最佳實踐

### Prompt 撰寫原則

1. **具體明確**：使用具體的描述詞，避免模糊表達
2. **分層結構**：基礎描述 → 細節特徵 → 風格效果
3. **負面提示**：明確列出不想要的元素
4. **權重控制**：使用 `(word:1.2)` 或 `[word]` 控制重要性
5. **一致性標記**：使用固定標記確保角色一致性

### 角色一致性保證

1. **固定描述詞**：為每個角色建立固定的描述詞庫
2. **種子管理**：使用固定 seed 範圍（如 1000-2000）
3. **Lora 使用**：為重要角色訓練專用 Lora
4. **後製標準化**：建立後製處理標準流程

### 批次處理策略

1. **模板化**：建立可重複使用的工作流程
2. **變數替換**：使用變數系統替換角色、場景
3. **自動化**：設計自動批次生成流程
4. **品質控制**：建立自動品質檢查機制

# Agent 路由快速參考指南

## 📋 概述

這個文件提供 Agent 自動路由系統的快速參考，幫助理解如何自動識別和使用對應的 AI Agent。

## 🎯 核心功能

當你在 Cursor 中提出問題時，系統會：
1. ✅ 自動分析你的問題關鍵詞
2. ✅ 匹配對應的 Agent
3. ✅ 讀取 Agent 的專業知識
4. ✅ 以角色標記回答（如 `[產品經理]`、`[前端工程師]`）

## 🎨 回答格式範例

### 單一 Agent 回答
```
[前端工程師] 我將為您建立一個現代化的 React 登入頁面...

[具體建議和代碼]
```

### 多 Agent 協作回答
```
[產品經理] 根據您的需求，我建議以下 6 天衝刺規劃...

[原型開發工程師] 同時建議先建立 MVP 原型來驗證核心功能...

[測試工程師] 建議在開發過程中同步編寫測試...
```

## 🚀 使用技巧

### 1. 明確描述需求
✅ **好的問題：** "我想建立一個 React 登入頁面，包含電子郵件和密碼驗證"
❌ **模糊問題：** "做個登入"

### 2. 提及關鍵技術
✅ **好的問題：** "用 React 建立一個待辦事項列表"
❌ **模糊問題：** "做個列表"

### 3. 說明使用場景
✅ **好的問題：** "規劃一個習慣追蹤 app 的 6 天衝刺，目標是 MVP"
❌ **模糊問題：** "規劃一下"

## 🔧 自定義和調整

如果需要調整路由規則：
1. 編輯 `.cursorrules` 文件
2. 修改關鍵詞映射
3. 調整角色名稱
4. 添加新的 Agent 映射

## 📚 相關文件

- `.cursorrules` - 完整的路由規則和配置
- 各 Agent 文件 - 詳細的專業知識和方法論

## 💬 反饋和改進

如果發現路由識別不準確：
1. 記錄問題場景
2. 檢查關鍵詞映射
3. 調整 `.cursorrules` 中的規則

**記住：** 你不需要手動指定 Agent，系統會自動識別並使用對應的專業知識！
